수동 작업(에디터에서 직접 클릭, 수동 파일 편집 등)은 절대 추천하지 말 것. 항상 CLI/스크립트/자동화 방법만 제시한다.

**Why:** 사용자가 명시적으로 요청. 수동 작업 옵션을 "빠른 방법"으로 제시했다가 거절당함. CLI print 출력에도 "이렇게 직접 해주세요" 류 메시지를 넣었다가 "규칙에 넣어"라고 지적당함.

**How to apply:**
- 어떤 작업이든 "에디터에서 직접", "수동으로", "직접 클릭" 같은 옵션은 선택지에서 제외.
- CLI 출력(print)에도 사용자가 손을 대야 하는 지시문("Project Settings > ...", "플러그인 활성화하세요" 등)을 포함하지 말 것.
- 필요한 설정은 코드/설정 파일에 자동 반영한다 (예: .uproject에 plugin 추가).
- 자동화가 더 오래 걸리더라도 그 방향만 제안한다.
