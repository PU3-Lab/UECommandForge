## 2. 이전 재리뷰 대비 상태

- `class_symbol` 중복 검사:
  - `GenerateSingleClass(..., false, ...)` 결과의 `class_symbol` 값을 `TargetClassSymbols`로 추적하고, 중복 시 `DUPLICATE_CLASS_SYMBOL` issue를 추가하도록 보강됐다.
  - `FGenerateCppClassBatchDuplicateClassSymbolTest` automation test 등록도 확인했다.
- 실패 클래스 partial cleanup:
  - 단건 생성 write failure 경로에서 header/source 삭제 실패와 `Build.cs` 복구 실패를 감지하고 `CLEANUP_FAILED`, `cleanup_failed` validation, `ChangedFiles`에 반영하도록 보강됐다.
- 롤백 실패 리포트:
  - batch rollback 실패 시 `rollback_failed=true`, `partial_changes_remaining=true`, `applied=true`, `changed_files`에 잔존 경로를 기록하도록 개선됐다.
- rollback 테스트:
  - apply 중 실패를 유도하는 directory blocker fixture와 `rollback_failed=false`, `partial_changes_remaining=false`, 실패 클래스 잔존 파일 부재 검증이 추가됐다.
- 정적 형식 문제:
  - 신규 파일 trailing whitespace는 재검증 결과 발견되지 않았다.

## 3. 주요 발견 사항

### MEDIUM: 대소문자만 다른 클래스명이 같은 파일로 충돌할 수 있음

- 위치: `GenerateCppClassBatchCommandlet.cpp:138`
- 내용:
  batch preflight의 경로 중복 검사는 `header_path`, `source_path` 문자열 원문을 그대로 `TSet<FString>`에 넣어 비교한다. 따라서 `MyActor.h`와 `myactor.h`처럼 대소문자만 다른 경로는 서로 다른 대상처럼 취급된다.
- 위험:
  macOS 및 Windows 기본 파일시스템처럼 대소문자 비구분 환경에서는 두 경로가 같은 파일을 가리킬 수 있다. 이 경우 dry-run preflight는 성공하지만 apply 중 두 번째 클래스가 `CPP_CLASS_FILE_EXISTS` 또는 파일 덮어쓰기/쓰기 실패로 지연 실패할 수 있어, batch preflight가 사전에 원자적으로 차단한다는 기대가 약해진다.
- 권장 조치:
  중복 검사에 사용하는 경로 key를 `FPaths::ConvertRelativePathToFull(...).Replace("\\", "/")` 후 대소문자 비구분 비교 기준으로 정규화한다. 최소한 batch preflight의 `TargetPaths`에는 lower-case canonical key를 저장하고, 대소문자만 다른 클래스명 테스트를 추가한다.

## 4. 확인된 개선 사항

- `DUPLICATE_CLASS_SYMBOL` 누락 문제는 해소된 것으로 판단한다.
- 실패 클래스 자체의 partial artifact cleanup 실패 감지는 단건 생성 helper에서 처리되도록 개선됐다.
- rollback 실패 시 표준 Result JSON 필드(`applied`, `changed_files`)만 보는 소비자가 잔존 변경을 확인할 수 있게 됐다.
- rollback validation 계약은 automation test에서 이전보다 명확히 고정됐다.
