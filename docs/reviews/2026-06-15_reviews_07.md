## 1. 재검증

### ✅ F5 — no-taint 회귀 테스트로 승격 확인

- 신규 `FDiffPlatformConfigEngineIniNotModifiedTest`:
  1. `<Project>/Config` 하위 **모든 `.ini`를 재귀 수집**해 실행 전 내용을 스냅샷(`OriginalContents`).
  2. **실제 `-Categories=Engine`**(원래 오염을 유발한 카테고리) + `-Platforms=Windows,Mac`로 `DiffPlatformConfig` 실행.
  3. 실행 후 각 ini를 다시 읽어 `TestEqual(CurrentContent, OriginalContent)`로 **바이트 동일** 단언.
- _06이 제안한 옵션 2(관찰→회귀 승격)를 정확히 구현했다. 향후 config 로드 경로가 소스 ini를 writeback하면 이 테스트가 즉시 FAIL → 재발 방지가 코드로 고정됐다.
- 평가: 구조적 read-only 가드 대신 **회귀 핀**으로 닫은 방식이지만, 결과적으로 동등한 보장을 제공한다. **F5 완전 해소로 인정**.

### 🟢 uasset/umap 잔존 — 원인 규명 완료

- 커밋 메시지가 `sample/Content/Tests/**`의 `.uasset`/`.umap` 미커밋 변경은 **타 빌더 테스트(Character, StateTree 등)의 고유 side effect이며 9A와 무관**함을 명시. _06의 잔여 항목이 규명·문서화됐다(9A 범위 밖, 별도 위생 과제로 남김).
