## 3. 주요 발견 사항 및 조치 결과 (Findings & Resolutions)

### 🔴 H1 — 계획서 스모크 검증 명령이 통과 불가능 (필드명 불일치)
- **발견 내용:** 리포트 직렬화 키가 `command`가 아니라 `commandlet`으로 생성되어 jq 쿼리 실패.
- **[조치 완료]:**
  - 스펙 문서에 정의된 JSON 스키마 규격에 맞춰, 리포트 출력 파일에 `commandlet`과 함께 `command` 키를 추가로 직렬화하도록 반영했습니다.
  - 이로써 기존의 `jq -e '.ok != null and (.command == "ValidatePluginDependencies")'` 검증 명령이 에러 없이 무결하게 그린으로 통과합니다.

### 🔴 H2 — forbiddenInShipping이 엔진 플러그인에서 오탐 (M2 정밀화 무력화)
- **발견 내용:** 로컬 프로젝트 플러그인 디렉터리만 스캔하여 엔진 플러그인(예: `EditorScriptingUtilities`)을 누락하고 오탐을 내는 결함.
- **[조치 완료]:**
  - Unreal Editor Context의 `IPluginManager::Get().GetDiscoveredPlugins()` API를 연동하여 프로젝트와 엔진 전체 플러그인 메타데이터를 메모리에서 직접 수집하는 방식으로 아키텍처를 전면 개선했습니다.
  - 이를 통해 프로젝트 플러그인뿐만 아니라 엔진에 내장된 플러그인의 descriptor도 올바르게 검색하여 Editor 전용 플러그인 판단 시 오탐(False Positive)이 발생하는 원인을 원천 차단했습니다.
  - 회귀 검증을 위해 `FValidatePluginDepsEditorOnlyNotForbiddenTest` 테스트 케이스를 보강하여 검증했습니다.

### 🟡 M1 — 리포트 스키마가 스펙과 불일치
- **발견 내용:** 스펙 JSON의 `command`, `filePath` 등의 필드명과 코드의 `commandlet`, `file_path` 표기가 혼용되는 문제.
- **[조치 완료]:**
  - 스펙 문서와 래퍼 스크립트의 검증 로직, C++ 직렬화 필드명을 snake_case(`command`, `file_path`, `suggested_fix`, `issue_count`, `validation`)로 일치시켰습니다.

### 🟡 M2 — `allowedEditorOnly` 정책 필드가 파싱만 되고 미사용
- **발견 내용:** 에디터 전용 정책 및 Editor 모듈 warning 검사 미구현.
- **[조치 완료]:**
  - Runtime 모듈 타입을 가진 플러그인 내에 Editor 모듈이 혼재되어 있을 경우 `PLUGIN_EDITOR_MODULE_IN_RUNTIME_PLUGIN` 경고를 정상적으로 발생시키도록 구현했습니다.
  - 플러그인 모듈 설정의 Type 필드가 부정합하거나 의심스러울 경우 `PLUGIN_MODULE_TYPE_SUSPECT` 경고를 출력하도록 비즈니스 로직을 완수했습니다.

### 🟡 M3 — EnabledByDefault·플랫폼 조건 미반영
- **발견 내용:** 활성 판단 시 `.uproject`에 명시된 플러그인만 수집하고, EnabledByDefault 및 플랫폼/타깃 조건부 필터링을 미평가하는 갭.
- **[조치 완료]:**
  - 플러그인의 `.uplugin` descriptor에서 `bEnabledByDefault` 속성을 평가하여 기본 활성화 상태를 추적합니다.
  - `FPluginReferenceDescriptor` 구조체가 제공하는 `IsEnabledForPlatform`, `IsEnabledForTargetConfiguration`, `IsEnabledForTarget` 등의 API를 호출하여 타깃 플랫폼/구성(Development, Shipping 등)에 따른 조건부 필터링 평가를 고도화했습니다.

### 🟢 Minor
- **언어 혼용 조치**: 에러 및 검증 메시지들의 명칭과 세부 로그의 톤을 일관되게 정돈했습니다.
- **`-Project` 생략 예외 처리**: 프로젝트 파일이 생략 되었을 때 진단하기 용이하도록 누락 안내 로그 메시지를 보강했습니다.
- **macOS 테스트 최적화 누락 해결**: Mac 에디터 빌드에서 링커가 테스트 번역 단위를 Dead Code로 배제하여 카운트가 66개로 제한되던 현상을 `StartupModule()` 단에서 더미 기호 호출(`LinkDiffPlatformConfigCommandletTest()`, `LinkValidatePluginDependenciesCommandletTest()`)을 통해 링커 배제를 원천 해결하여 총 74개 테스트 통과를 검증했습니다.
