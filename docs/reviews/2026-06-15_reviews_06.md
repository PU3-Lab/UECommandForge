## 2. 발견 사항

### ✅ F6 확인 — editor-module warning 폴백

- warning 루프가 `EnabledPlugins`에 없는 플러그인에 대해 descriptor 기반 `EnabledByDefault == Enabled && SupportsTargetPlatform` 폴백 평가(`cpp:384-403`). `.uproject`/정책에 없는 enabled-by-default 인트리 플러그인의 경고 누락 사각지대가 해소됐다.

### ✅ F7 확인 — Unspecified 엔진 플러그인 보정 + 회귀 테스트

- `Desc.EnabledByDefault == Unspecified`이고 descriptor 경로가 엔진 디렉터리 하위면 `bEnabled = true`로 보정(`cpp:194-204`, `FullDescPath.StartsWith(FullEngineDir)`로 엔진 한정). 프로젝트 Unspecified 플러그인은 영향 없음.
- 회귀 테스트 신규: `FValidatePluginDepsEngineUnspecifiedEnabledTest`가 `-TestEnginePluginsDir=`에 Runtime 모듈·Unspecified 기본값의 가짜 엔진 플러그인을 두고 required 검증이 통과함을 단언.
- 주의(설계 판단): "Unspecified == enabled"는 엔진 플러그인 다수에 맞는 **근사**다. 일부 Unspecified-but-disabled 엔진 플러그인을 활성으로 오인할 수 있으나, required 검증에서는 false-negative를 줄이는 방향이라 수용 가능.

### ⚠️ F5 — 구조적 수정이 아니라 관찰 기반 해결

- 커밋 메시지·[review_fixes_report.md](2026-06-15_review_fixes_report.md)는 "git status 확인으로 오염이 전혀 발생하지 않음을 실증" → 완전 PASS 전환을 선언한다.
- 그러나 `DiffPlatformConfigCommandlet.cpp` 변경(+89줄)은 **배열 비교 관련**이고, **config writeback을 막는 코드(NoSave/SetReadOnly/Dirty 제어/GConfig 분리 등)는 0건**이다(diff grep 확인).
- 즉 F5는 (1) 오염된 `sample/Config/*Engine.ini`를 복원하고 (2) 재실행 시 오염이 재현되지 않음을 관찰한 것이다. 원래 오염이 `FConfigFile`의 비결정적 dirty→flush 부작용이었다면, **코드 보장 없이 "이번엔 안 났다"는 재발 가능성을 닫지 못한다.**
- 보완 권장:
  1. `ReadIntoLocalFile` 경로가 소스 ini를 절대 dirty/flush하지 않음을 코드로 보장(로컬 `FConfigFile` 사용이 writeback과 분리됨을 명시적 단언/주석, 필요 시 read-only 가드).
  2. 또는 "오염 미발생"을 **자동화 테스트**로 고정(커맨드렛 실행 전후 `sample/Config/*Engine.ini` 해시 동일 단언) — 관찰을 회귀로 승격.

### 🟢 잔여 — 작업 트리 uasset/umap 수정 지속

- `git status`에 `sample/Content/Tests/**`의 `.uasset`/`.umap`(BP_TestCharacter, TestMap 등) 수정이 여전히 미커밋으로 남아 있다(세션 시작 시점부터 존재). Engine.ini와 별개의 자동화 테스트 부산물로 보이나, 커밋 위생상 의도 여부 확인 권장. 9A 범위 밖.
