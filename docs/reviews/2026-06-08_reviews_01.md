## 2. 사용 가능한 언리얼 자동화 기능 목록

UECommandForge는 "LLM이 Spec JSON을 설계하고, C++ Commandlet은 빌더/검증기를 통해 이를 수행한다"는 구조적 방향을 따르며, 현재 다음 기능들이 지원되고 있다.

### 2.1 프로젝트 및 에셋 수명 주기 관리 (Asset Lifecycle)
*   **표준 폴더 구조 생성:** 프로젝트 표준 디렉터리 구성 자동화 (`create_project_folders.sh`)
*   **에셋 변경 계획 및 Preflight:** 변경 사항 적용 전 영향도 정적 분석 (`plan_asset_changes.sh`)
*   **스냅샷 및 트랜잭션 롤백:** 작업 도중 에러나 검증 실패 시 이전 에셋 스냅샷 상태로 롤백 수행 (`snapshot_assets.sh`, `apply_asset_changes.sh`, `rollback_asset_changes.sh`)

### 2.2 블루프린트 생성 및 제어 (Blueprint Generation)
*   **범용 블루프린트 생성:** 부모 클래스 유효성 및 덮어쓰기 옵션(`bAllowReplace`)을 반영한 안전한 생성 (`create_blueprint.sh`)
*   **캐릭터 및 AIController 생성:** AI Flow 구성을 위한 전용 Blueprint 에셋 빌드 (`create_character_bp.sh`, `create_ai_controller.sh`)
*   **디폴트 프로퍼티/컴포넌트 설정:** 블루프린트의 기본 프로퍼티 값 수정 및 컴포넌트 추가 (`set_blueprint_defaults.sh`)
*   **일괄 컴파일:** 블루프린트 일괄 무결성 검증 및 컴파일 수행 (`compile_blueprints.sh`)

### 2.3 C++ 클래스 생성 및 코드 정적 검증 (C++ & Verification)
*   **C++ 클래스 자동 생성:** 언리얼 C++ 클래스 템플릿 코드 및 리플렉션 매크로 자동 작성 (`generate_cpp_class.sh`)
*   **UHT 로그 분석:** UHT 컴파일 에러 발생 시 원인이 되는 매크로 누락 분석 리포트 (`analyze_uht_log.sh`)
*   **C++ 리플렉션 및 Build.cs 검증:** 필수 매크로(`UPROPERTY`, `UFUNCTION`) 누락 및 모듈 의존성 적합성 검사 (`validate_cpp_reflection.sh`, `validate_buildcs.sh`)

### 2.4 StateTree 기반 AI Flow 및 맵 배치 (AI Flow & Placement)
*   **StateTree 생성 및 구성:** UE 5.7 StateTree Editor API를 사용해 State, Task, Transition 연결 구조 자동 생성 (`create_statetree.sh`)
*   **AI 플로우 바인딩:** Character와 AIController를 바인딩하고 리플렉션으로 StateTree 컴포넌트 참조 셋업 (`bind_ai_flow.sh`)
*   **AI 무결성 검증:** 캐릭터-컨트롤러-StateTree의 3방향 참조 구조 및 포제스 옵션 정합성 진단 (`validate_ai_flow.sh`)
*   **원스톱 AI 플로우 워크플로우:** 위의 전체 단계를 하나의 JSON 스펙 파일로 일괄 수행 (`create_ai_flow.sh`)
*   **액터 맵 배치:** 지정된 Level/Map에 액터를 중복 없이 배치하고 중복 인스턴스 자동 정리 (`place_actor.sh`)

### 2.5 데이터 소스 관리 및 통합 검증 (Data & Validation Gate)
*   **데이터 테이블 임포트 및 검증:** 외부 JSON/CSV 데이터 스키마 유효성을 사전 진단하고 DataTable 에셋으로 임포트 (`import_data_source.sh`, `validate_data_source.sh`, `validate_datatable.sh`)
*   **프로젝트 환경설정 검증:** `DefaultEngine.ini` 등의 엔진 구성 파일 정합성 진단 (`validate_config_rules.sh`)
*   **통합 품질 게이트:** 전체 프로젝트의 규칙 준수 상태를 종합 검사하여 Markdown/JSON 보고서 발행 (`validate_project_rules.sh` 또는 `PrototypeAutomation`)

## 3. Codex 작업 시간 지연 진단 및 해결 방안

Codex 에이전트 환경에서 C++ 클래스를 생성하고 블루프린트에 메쉬 컴포넌트를 할당하는 등의 작업을 수행할 때 소요 시간이 과도하게 길어지는 이슈에 대한 분석 및 최적화 방안이다.

### 3.1 원인 분석
1.  **UnrealEditor-Cmd 기동 비용 (핵심 병목):**
    현재 CLI 래퍼들은 각 명령 단위로 에디터를 백그라운드(`-nullrhi` 헤드리스)로 켠다. `generate_cpp_class.sh` 실행 완료 후, `create_blueprint.sh`를 구동하고, 이어서 `set_blueprint_defaults.sh`를 별도 명령으로 호출하는 **Atomic 방식의 순차 실행** 시 매 단계마다 언리얼 에디터의 기동/종료 시간(최소 20~40초)이 중복 누적된다.
2.  **C++ 컴파일 오버헤드:**
    새로운 C++ 클래스를 추가하면 에디터 모듈을 다시 컴파일하여 반영해야 하므로 Unreal Build Tool(UBT)에 의한 빌드 타임이 추가된다.

### 3.2 최적화 및 해결 방안

#### [방안 1] Workflow Commandlet 기반 배치(Batch) 처리 고도화
*   **원리:** 여러 개의 작업을 여러 번의 CLI 호출로 쪼개지 않고, 하나의 Spec JSON 파일에 전체 로직(C++ 클래스 정보 + 생성할 블루프린트 명세 + 부착할 컴포넌트 정보)을 계층 구조로 선언하여 **단 한 번의 에디터 기동으로 완료**시킨다.
*   **적용:** `setup_npc_character.sh`와 같이 여러 역할을 묶어 처리하도록 구현된 Workflow Commandlet을 생성하여 배치 처리한다.

#### [방안 2] Blueprint-Only 프로토타이핑 활용
*   **원리:** C++ 클래스 추가가 구조적으로 필수적이지 않은 경우, Unreal 엔진의 기본 클래스(Actor, Character 등)를 부모로 삼아 블루프린트를 직접 생성하고 컴포넌트를 부착한다.
*   **효과:** C++ 코드 빌드 및 핫 리로드 시간을 아예 생략할 수 있어 실행 효율이 획기적으로 개선된다.

#### [방안 3] Unreal Commandlet 실행 플래그 추가 최적화
*   **원리:** `run_commandlet.sh` 내 에디터 CLI 파라미터에 초기화 생략 플래그를 추가로 전달해 기동 시간을 단축한다.
*   **추천 플래그:**
    *   `-NoShaderCompile` / `-SkipShaderCompile`: 백그라운드 셰이더 컴파일 대기를 생략한다.
    *   `-NoSound`: 오디오 서브시스템 기동을 생략한다.
    *   `-NoPhysXVehicles` / `-DisablePhysicsCoSim`: 필요 없는 물리 엔진 서브시스템 모듈 로드를 제한한다.

#### [방안 4] 증분 빌드 및 라이브 코딩 최적화
*   **원리:** C++ 컴파일 시 전체 모듈을 다시 빌드하지 않도록 에셋 계획 시스템을 고도화하고, UBT가 증분 빌드(Incremental Build)로 코드 변경분만 빠르게 컴파일하여 핫 리로드 시간을 최소화한다.
