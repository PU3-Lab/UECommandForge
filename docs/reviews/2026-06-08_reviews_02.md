## findings

### CRITICAL

### HIGH

**H1. 검증되지 않은 플래그가 전체 commandlet에 무조건 적용됨**
`tools/ue/run_commandlet.sh` `run_unreal_commandlet()`

```bash
-NoShaderCompile -SkipShaderCompile -NoSound -NoPhysXVehicles -DisablePhysicsCoSim
```

- 플랜 §2.1은 `-DisablePhysicsCoSim`(비표준/미문서화), `-NoPhysXVehicles`(UE 5.x에서 PhysX Vehicle 제거됨)를 **검증 전 채택 금지**로 명시했으나 검증 없이 도입됨.
- 플래그가 함수 단일 지점에 박혀 있어 30개 이상 **모든** commandlet에 무조건 적용됨. `-NoShaderCompile`이 셰이더/머티리얼이 필요한 작업(일부 블루프린트 컴파일 등)을 깨뜨릴 위험.
- **수정안**: 무효 플래그 2종 제거 후 UE 5.7에서 1개씩 실측 검증. 셰이더 의존 commandlet과의 충돌 확인. 필요 시 commandlet별 플래그 프로파일 분리.

**H2. 트랜잭션/롤백 미구현 — 부분 실패 시 에셋 상태 일관성 붕괴**
`CreateBlueprintBatchCommandlet.cpp:68-160`

- 플랜 §4.2는 "중간 실패 시 snapshot/rollback 연동, 부분 성공 상태를 남기지 않음"을 요구. 현재는 item 실패 시 `continue`만 하고 성공 item은 이미 디스크에 Save됨.
- item 3개 중 2번 실패 시 1번 저장/2번 부분/3번 저장 상태로 남음. `Report.TransactionId`는 기록만 될 뿐 트랜잭션 의미 없음.
- **수정안**: 배치 시작 전 snapshot, `bOk == false` 시 전체 rollback. 또는 부분 성공 정책을 의도적으로 선언하고 명시 문서화.

**H3. 신규 파서/Commandlet 테스트 전무**

- `Tests/`에 형제 commandlet마다 `*CommandletTest.cpp`가 존재(현재 53개 통과)하나 `BlueprintBatchSpecParser`·`CreateBlueprintBatchCommandlet`만 테스트 없음. 프로젝트 컨벤션 및 testing.md(80%/TDD) 위반.
- **수정안**: 파서 단위 테스트(kind 불일치, items 누락, 필수 필드 누락, 정상 파싱) + 배치 부분 실패/롤백 케이스 추가.

### MEDIUM

**M2. `version` 필드 미검증**
`BlueprintBatchSpecParser.cpp:24`

- `version`을 읽지만 값 검증 없음. `kind`는 검증하므로 일관성 차원에서 version 검증 권장.

## 잘된 점

- PERF 로깅이 stderr로 출력되어 stdout(OUTPUT_JSON 경로 반환) 오염 없음. `local exit_code=$?` 캡처 정확.
- 파서가 에러를 누적 수집(`OutErrors.Num() == InitialErrorCount` 패턴)하여 다중 검증 오류를 한 번에 보고 — 견고한 입력 검증.
- 배치 내 컴포넌트 변경 시에만 중간 컴파일, 변경 없으면 Save 생략(`bChanged`)으로 불필요한 I/O 회피.
