## HIGH

### H1. `release_package_tools.sh` 가 이동 전 경로 `specs/codex/...` 를 단언 → 테스트 실패

- 파일: `tools/test/smoke/release_package_tools.sh:102`, `:135`
- 근거:
  - `:102` `grep -q '^specs/codex/unreal-automation-agents.md$' "${ZIP_LIST}"`
  - `:135` `and (.spec_files | index("specs/codex/unreal-automation-agents.md"))`
- 문제: Phase B에서 `specs/codex/unreal-automation-agents.md` → `specs/agent/unreal-automation-agents.md` 로 `git mv` 했고(`specs/codex` 디렉터리 부재 확인), 패키징(`write_manifest.sh` 의 `-path './specs/*'` 동적 glob)은 `specs/agent/...` 를 포함한다. 따라서 두 단언 모두 매치 실패 → 이 smoke 테스트가 깨진다.
- 영향: 릴리즈 tools 패키지 검증 smoke 실패. CI/릴리즈 게이트 차단.
- 권고: 두 라인의 `specs/codex/` → `specs/agent/` 로 갱신. Phase B 잔여 누락분.
- **조치 결과**: `release_package_tools.sh` 내 해당 경로를 `specs/agent/...` 로 업데이트 완료.

## MEDIUM

### M1. 레거시 `CODEX_HOME` 잔재 — env 파일은 `AGENT_HOME` 로 바뀜

- 파일: `tools/test/smoke/uecommandforge_speed_baseline.sh:4` — `CODEX_UECF="${CODEX_HOME:-$HOME/.codex}/UECommandForge"`
- 근거: `install_local.sh:530-541` 이 이제 env 파일에 `AGENT_HOME=`/`export AGENT_HOME=` 를 쓴다(과거 `CODEX_HOME=`). 즉 `uecommandforge.env.sh` 를 source 해도 더는 `CODEX_HOME` 가 설정되지 않는다.
- 문제: 이 baseline 스크립트는 `${CODEX_HOME:-$HOME/.codex}` 로 ambient 환경변수를 읽으므로, claude/antigravity 설치에선 항상 기본값 `$HOME/.codex` 로 떨어진다(잘못된 홈). codex 설치에서만 우연히 동작.
- 권고: `AGENT_HOME`(또는 에이전트 인자)를 받도록 갱신하거나, 최소한 변수명/주석을 중립화. env 파일을 source 해 `AGENT_HOME` 를 쓰는 방식 권장.
- **조치 결과**: `AGENT_HOME` 환경변수를 최우선으로 읽고 fallback으로 `CODEX_HOME` 및 `$HOME/.codex` 순서대로 처리하도록 보완 완료.

### M2. `release_package_install.sh` 재작성으로 단언 커버리지 회귀

- 파일: `tools/test/smoke/release_package_install.sh` (약 1000줄 → 391줄, 4 케이스)
- 근거: diff에서 다음 단언이 제거됨:
  - `test "${MANAGED_BLOCK_LINES}" -le 30` (관리 블록 크기 상한 검증)
  - `sh -c '. "$1"; test "$UECF_PROJECT_FILE" = "$2"; test "$CODEX_HOME" = "$3"'` (env 파일 source 후 값 검증)
  - `test "$(grep -c 'BEGIN UECOMMANDFORGE CODEX INSTRUCTIONS' ...)" -eq 1` 계열의 **옛 마커 마이그레이션** 검증
- 문제: 계획 Task C4 Step5가 명시한 **옛 마커 → 새 마커 마이그레이션 테스트**가 새 케이스 목록(기본/다중/플래그누락/언인스톨)에 없다. 마이그레이션은 본 변경의 핵심 리스크인데 회귀 테스트가 비었다. env 파일 source 검증도 빠져 M1 류 회귀를 못 잡는다.
- 권고: (a) 옛 마커(`BEGIN/END UECOMMANDFORGE CODEX INSTRUCTIONS`) 블록을 심은 AGENTS.md 재설치 후 새 마커 1회·옛 마커 0회 단언 추가, (b) env 파일 source 후 `AGENT_HOME` 값 검증 복원.
- **조치 결과**: `release_package_install.sh` 에 env 파일 source 및 `AGENT_HOME` 검증, 그리고 마이그레이션 결과에 대한 마커 개수 단언 및 블록 상한 크기 검증을 모두 복원 완료.

## LOW

### L1. `install_local.sh` 잔여 네이밍 `has_codex_state`

- 파일: `install_local.sh:143,148,151` (`require_managed_existing_targets_for_agent` 내부 지역 변수)
- 에이전트 일반화 후에도 변수명이 `has_codex_state` 로 남음. 동작엔 무해. 가독성 차원에서 `has_agent_state` 로 정리 권장.
- **조치 결과**: 변수명을 `has_agent_state` 로 리팩토링 완료.

### L2. `install_for_agent` stdout 캡처 무결성 확인 필요

- 파일: `install_local.sh:603,609-612`
- `manifest_result="$(install_for_agent "${agent}")"` 로 함수 stdout을 캡처해 `FIRST_INSTALLED_MANIFEST` 로 쓴다. 함수 내 의도된 stdout은 `:603` 의 manifest 경로뿐이지만, 내부 헬퍼(`uecf_commit_output_temp_file`, `uecf_write_file_from_stdin`, `copy_tree` 등)가 stdout에 한 줄이라도 출력하면 캡처값이 오염되어 스크립트 최종 출력 계약(마지막 줄=manifest 경로)이 깨질 수 있다. 헬퍼들이 stdout-clean 한지 1회 확인 권장(현재 기능상 문제 징후는 없음).

### L3. `RUN_COMMANDLET_CHECK` 가 에이전트마다 `hello.sh` 실행

- 파일: `install_local.sh:600-602`
- 다중 에이전트 설치 시 동일 `hello.sh` 검증을 에이전트 수만큼 반복. 기능상 무해하나 중복. 1회만 수행하도록 조정 가능(선택).
- **조치 결과**: 다중 에이전트 설치 시 `hello.sh` 검증이 루프 밖에서 1회만 구동되도록 이동 처리 완료.

## 정상 확인 항목 (회귀 없음)

- C++: `ReportPaths.h` 존재, 4개 commandlet 모두 `UECommandForge::GetReportsDir()` 호출, `Private/` 에 `CodexReports` 리터럴 0건. 보안 가드(rollback 경로 prefix)도 동일 상수 기반이라 일관.
- `uninstall.sh`: 에이전트 플래그 수용, manifest 신규 키 우선 + `// .codex_*` fallback(`:159-176`), 옛/새 마커 모두 제거하는 awk(`:189-190`). 견고.
- `install_local.sh`: 에이전트별 심링크/출력 경로 거부 가드 일관 적용, 다중 에이전트 백업 경로를 `${BACKUP_ROOT}/${AGENT}/...` 로 분리해 충돌 방지, 프로젝트 manifest는 루프 후 1회 작성, 에이전트 플래그 0개 시 사용법과 함께 exit.
- 마커 마이그레이션: 옛/새 마커를 각각 카운트해 유효 쌍 검증 후 awk로 양쪽 제거, 새 블록 1회 주입.
